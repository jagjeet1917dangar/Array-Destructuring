# 1. JavaScript â€“ Ways to Create Functions

---

## 1. Function Declaration

```js
function add(a, b) {
    return a + b;
}
```

* ye sabse basic aur normal way hai function banane ka
* yaha **hoisting hoti hai**, matlab function ko define se pehle bhi call kar sakte ho
* Example:

```js
console.log(add(2, 3)); // 5
function add(a, b) { return a + b; }
```

---

## 2. Function Expression

```js
const multiply = function(x, y) {
    return x * y;
};
```

* isme function ko ek variable me assign karte hain
* **hoisting nahi hoti**, so call karne se pehle define karna zaruri hai
* mostly anonymous functions ke liye use hota hai

---

## 3. Arrow Function (ES6)

```js
const sub = (a, b) => a - b;
```

* ye new and short syntax hai
* single line me likh sakte ho bina `return` likhe
* but arrow function ka apna `this` nahi hota, isliye object ke andar dikkat de sakta hai
* Example:

```js
const greet = name => console.log("Hello " + name);
```

---

## 4. Anonymous Function

* ye function ka koi naam nahi hota
* mostly callback ke form me use hota hai

```js
setTimeout(function() {
    console.log("Timeâ€™s up");
}, 2000);
```

---

## 5. Immediately Invoked Function Expression (IIFE)

```js
(function() {
    console.log("Runs instantly");
})();
```

* ye function turant execute ho jaata hai jaise hi define hota hai
* pehle ke time me **private scope** banane ke liye use karte the

---

### Thoda sa summary:

* Function declaration â€“ normal and hoisted
* Function expression â€“ variable ke through, no hoisting
* Arrow function â€“ short syntax, no `this`
* Anonymous â€“ mostly callback ke liye
* IIFE â€“ execute hote hi run

---

**Note:** sirf syntax nahi, kab aur kyu use karna hai wo bhi samajhna zaruri hai.

# 2. JavaScript â€“ Function Parameters and Arguments

---

## 1. Function Parameters

* Parameters wo variables hote hain jo hum function ke definition me likhte hain.
* ye basically placeholders hote hain values ke liye.
* Example:

```js
function add(a, b) {
    return a + b;
}
```

* yaha `a` aur `b` **parameters** hain.
* jab function call hoga tab ye values receive karenge.

---

## 2. Function Arguments

* Arguments wo values hain jo function call karte time pass ki jaati hain.
* Example:

```js
add(5, 10);
```

* yaha `5` aur `10` **arguments** hain jo `a` aur `b` ko milenge.

---

## 3. Default Parameters (ES6)

* kabhi kabhi value pass nahi hoti, tab hum default value de sakte hain.

```js
function greet(name = "Guest") {
    console.log("Hello " + name);
}
greet(); // output: Hello Guest
greet("Jagjeet"); // output: Hello Jagjeet
```

* ye kaafi useful hota hai jab optional parameters ho.

---

## 4. Rest Parameters (...)

* jab exact number of arguments nahi pata ho, tab rest parameters use karte hain.

```js
function sum(...nums) {
    let total = 0;
    for (let n of nums) {
        total += n;
    }
    return total;
}
console.log(sum(1, 2, 3, 4)); // 10
```

* ye saare arguments ko ek array ke form me store karta hai.
* matlab jitne bhi arguments pass karo, sab handle ho jaayenge.

---

## 5. Arguments Object (old style)

* function ke andar ek **arguments** naam ka object milta hai jo sab passed values rakhta hai.

```js
function show() {
    console.log(arguments);
}
show(1, 2, 3);
```

* ye array jaisa lagta hai but actually array nahi hota (array-like object).
* modern JS me `rest parameters` zyada use hote hain instead of `arguments`.

---

### Thoda sa recap:

* Parameters â†’ likhe jaate hain function ke definition me
* Arguments â†’ actual values jo call time par pass hoti hain
* Default parameter â†’ jab koi argument missing ho
* Rest parameter â†’ unknown number of arguments handle karne ke liye
* `arguments` object â†’ purana tareeka, ab kam use hota hai

---

**Note:** Function ke parameters aur arguments me confusion mat rakhna, dono related hain but same nahi hote.

# 3. JavaScript â€“ Scope of Function

---

## 1. What is Scope?

* Scope ka matlab hai **kahaan tak ek variable accessible hai**.
* Basically, ek variable function ke andar ya bahar kaha tak valid hai, ye scope decide karta hai.
* JS me mainly 3 types ke scopes hote hain.

---

## 2. Global Scope

* Jo variable function ke bahar declare hota hai wo global scope me hota hai.
* usse code ke kisi bhi part me use kar sakte ho.

```js
let a = 10;

function show() {
    console.log(a); // accessible here
}
show();
console.log(a); // accessible here also
```

* par dikkat ye hoti hai ki global variable ko kahi bhi change kiya ja sakta hai, so risky hota hai.

---

## 3. Local / Function Scope

* jo variable function ke andar declare hota hai wo **sirf us function ke andar** hi access hota hai.

```js
function test() {
    let x = 5;
    console.log(x); // works fine
}
test();
console.log(x); // error, x is not defined
```

* matlab bahar se access nahi kar sakte.
* ye clean code ke liye best practice hai.

---

## 4. Block Scope (introduced in ES6)

* `let` aur `const` ke variables sirf us block `{ }` ke andar hi accessible hote hain.

```js
{
    let y = 20;
    const z = 30;
    console.log(y, z); // works
}
console.log(y); // error
```

* pehle `var` ka koi block scope nahi hota tha, sirf function scope hota tha.

---

## 5. Lexical Scope

* Lexical scope ka matlab hai ki **inner function** apne parent function ke variables ko access kar sakta hai.
* Example:

```js
function outer() {
    let a = 10;
    function inner() {
        console.log(a); // accessible due to lexical scope
    }
    inner();
}
outer();
```

* but parent function child ke variables ko access nahi kar sakta.

---

## 6. Nested Function Scope

* jab ek function ke andar dusra function hota hai, to wo parent ke variables ko access kar sakta hai, par parent uske nahi.

```js
function main() {
    let msg = "Hello";
    function sub() {
        console.log(msg);
    }
    sub();
}
main();
```

* is concept ke base pe **closures** ka idea aata hai (next topic me milta hai ye).

---

### Short Recap:

* Global scope â†’ sab jagah accessible
* Function scope â†’ sirf function ke andar
* Block scope â†’ sirf `{}` ke andar
* Lexical scope â†’ inner function can access parent variables

---

**Note:** Scope samajhna zaruri hai kyunki galat variable access se code me unexpected bugs aa jaate hain.

# 4. JavaScript â€“ Hoisting in Detail (with Execution Context)

---

## 1. What is Hoisting?

* Hoisting ek aisa concept hai jisme **JS code run hone se pehle** variables aur functions ko memory me allocate kar diya jaata hai.
* Matlab, aap unhe likhne se pehle bhi access kar sakte ho (but behavior depend karta hai `var`, `let`, `const`, or `function` pe).

Simple words me â€” JS code run hone ke do phase hote hain:

1. **Memory Creation Phase (Hoisting hoti hai yaha)**
2. **Execution Phase (code actually chalta hai)**

---

## 2. Execution Context

* Jab bhi koi JS code run hota hai, ek environment create hota hai jisme variables aur functions store hote hain â€” usse kehte hain **Execution Context (EC)**.
* 2 types ke execution context hote hain:

  * **Global Execution Context (GEC)** â†’ code ke start me banta hai
  * **Function Execution Context (FEC)** â†’ jab bhi function call hota hai

Example visual socho:

```
Memory Phase:
var a = undefined
function test() { ... }

Execution Phase:
a = 10
test()
```

---

## 3. Hoisting with `var`

```js
console.log(a);  // undefined
var a = 10;
```

* JS ne `a` ko memory me reserve kar liya hota hai pehle se, par value assign nahi hoti (sirf `undefined`).
* isliye error nahi aata, but value `undefined` milti hai.

---

## 4. Hoisting with `let` and `const`

```js
console.log(b); // ReferenceError
let b = 5;
```

* `let` aur `const` bhi hoist hote hain, **but unhe access karne se pehle error aata hai**.
* ye period ko **Temporal Dead Zone (TDZ)** kehte hain.
* TDZ tab tak hota hai jab tak variable initialize nahi hota.

---

## 5. Hoisting with Functions

```js
sayHello();

function sayHello() {
    console.log("Hello World");
}
```

* Function declarations fully hoist hote hain.
* Matlab poora function memory me chala jaata hai, isliye use pehle call kar sakte ho.

But agar function ko expression ke form me likho:

```js
greet(); // error

var greet = function() {
    console.log("Hi there");
}
```

* tab ye sirf variable ke jaise hoist hota hai (value undefined hoti hai), function body nahi.

---

## 6. Step-by-step Example

```js
console.log(a);
var a = 10;
function show() {
    console.log("Inside function");
}
show();
```

**Behind the scenes (JS engine working):**

**Memory Creation Phase:**

```
a â†’ undefined
show â†’ function reference
```

**Execution Phase:**

```
console.log(a) â†’ undefined
a = 10
show() â†’ executes the function
```

Output:

```
undefined
Inside function
```

---

## 7. Important Points:

* Hoisting sirf **declarations** pe apply hota hai, **initialization** pe nahi.
* `var` â†’ hoisted as undefined
* `let` & `const` â†’ hoisted but not accessible (TDZ)
* Function declarations â†’ fully hoisted
* Function expressions & arrow functions â†’ act like variables (undefined until initialization)
